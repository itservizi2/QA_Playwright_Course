## Exerciții TypeScript - Tabel Complet

| Secțiune | Recapitulare | Exercitiu | Cerință Completă |
| :-- | :-- | :-- | :-- |
| **1. Modificatori de acces (public, protected, private)** | public — accesibil oriunde. protected — accesibil in clasa curenta si in subclase. private — accesibil doar in clasa in care a fost definit. | **Exercitiul 1.1 — BankAccount** | Creati o clasa BankAccount cu proprietatea publica name (numele titularului) si proprietatea privata balance (soldul, initial 0). Adaugati doua metode publice: deposit(amount) care adauga suma la balance, si getBalance() care returneaza balance. Creati un obiect, faceti doua depuneri si afisati soldul. |
|  |  | **Exercitiul 1.2 — Person \& Student** | Creati o clasa Person cu name (public) si age (protected). Apoi clasa Student care extinde Person, primeste in plus studentId (private), si are o metoda publica showInfo() care afiseaza numele, varsta si studentId. Creati un obiect Student si apelati showInfo(). |
| **2. Mostenire si suprascrierea metodelor (inheritance \& overriding)** | cu extends o clasa copil mosteneste proprietatile si metodele clasei parinte. O metoda din copil cu aceeasi semnatura ca in parinte o suprascrie (override). | **Exercitiul 2.1 — Animal, Dog, Cat** | Creati o clasa Animal cu proprietatea name si metoda sound() care afiseaza "Some sound...". Creati apoi doua subclase: Dog si Cat. Suprascrieti sound() in fiecare astfel incat Dog sa afiseze "Woof!", iar Cat "Meow!". Creati cate un obiect din fiecare si apelati sound(). |
|  |  | **Exercitiul 2.2 — Shape, Rectangle, Circle** | Creati o clasa Shape cu metoda area() care returneaza 0. Creati Rectangle (cu width, height) si Circle (cu radius), care suprascriu area() pentru a calcula corect aria (folositi Math.PI pentru cerc). Afisati aria pentru un dreptunghi 4×5 si un cerc cu raza 3. |
| **3. Interfete** | o interfata descrie forma unui obiect — proprietati, proprietati optionale (?), proprietati readonly si semnaturi de metode. Clasele pot implementa o interfata cu implements. | **Exercitiul 3.1 — Product** | Definiti interfata Product cu name (string), price (number) si description (string, optional). Creati doua obiecte de tip Product: unul cu descriere, unul fara. Afisati-le pe ambele. |
|  |  | **Exercitiul 3.2 — Vehicle \& Car** | Definiti interfata Vehicle cu vin (readonly string), brand (string) si metoda start(): void. Creati clasa Car care implementeaza Vehicle. Creati un obiect Car si apelati start(). Incercati sa modificati vin dupa creare si observati eroarea TypeScript (lasati linia respectiva comentata). |
| **4. Module (export / import)** | cu export facem disponibile elemente dintr-un fisier; cu import le folosim in alt fisier. Doua forme uzuale: named import si namespace import (import * as ...). | **Exercitiul 4.1 — MathUtils.ts (named import)** | Creati fisierul MathUtils.ts care exporta: constanta PI = 3.14, functia square(x: number): number si clasa Calculator cu metoda statica add(a: number, b: number): number. Intr-un fisier App.ts faceti named import pentru toate trei si folositi-le. |
|  |  | **Exercitiul 4.2 — Acelasi modul, namespace import** | Folosind acelasi fisier MathUtils.ts, in App.ts faceti namespace import: import * as math from "./MathUtils". Folositi math.PI, math.square si math.Calculator.add. |
| **5. Constructor \& method overloading** | in TypeScript declarati mai multe semnaturi pentru acelasi constructor sau metoda, urmate de o singura implementare care le acopera pe toate. | **Exercitiul 5.1 — Rectangle (constructor overloading)** | Creati clasa Rectangle cu doua semnaturi de constructor: una fara parametri (creeaza un patrat 1×1) si una cu width si height. Adaugati metoda area() care returneaza width × height. Creati cate un obiect din fiecare forma si afisati aria. |
|  |  | **Exercitiul 5.2 — Greeter (method overloading)** | Creati clasa Greeter cu metoda greet supraincarcata: greet() → "Hello!"; greet(name) → "Hello, <name>!"; greet(name, lang) → daca lang === "fr" returneaza "Bonjour, <name>!", altfel "Hello, <name>!". Apelati toate cele trei variante. |
